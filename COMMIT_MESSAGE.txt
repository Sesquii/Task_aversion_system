fix(ui): remove incorrect 0-10 to 0-100 scaling in completion page

Fixed critical bug where actual values were being incorrectly scaled by 10x
when displayed on the completion page, causing sliders to show 10x the 
initialization values.

## Problem

The completion page (complete_task.py) was applying scaling logic that multiplied
values by 10 if they were <= 10, attempting to convert from 0-10 to 0-100 scale.
However, values are already stored on the 0-100 scale from initialization, so this
caused actual values to be incorrectly scaled to 10x their initialization values.

## Fixes Applied

### 1. Removed Scaling Logic from UI (complete_task.py)

- Removed scaling from `get_default_value()` function - now uses values as-is
- Removed scaling from `current_expected_aversion` processing
- Removed scaling from all display code for "Initialized" labels:
  - Mental Energy (expected_mental_energy)
  - Task Difficulty (expected_difficulty)
  - Emotional Load (expected_emotional_load)
  - Physical Load (expected_physical_load)
- All sliders now correctly display values on 0-100 scale without modification

### 2. Data Correction Script (fix_scaled_actual_values.py)

Created script to retroactively fix existing data where actual values were
incorrectly scaled:

- Scans all completed instances for values that are approximately 10x predicted
- Only fixes values > 100 (suspiciously high, indicating incorrect scaling)
- Uses ratio tolerance (7.5x-12.5x) to detect incorrectly scaled values
- Divides incorrectly scaled values by 10 to correct them
- Works with both CSV and database backends
- Efficient CSV saving (batches updates, saves once at end)

**Execution Results:**
- Scanned 228 completed instances
- Fixed 11 instances with incorrectly scaled values
- Examples: Mental Energy 40.0 → 4.0, Emotional 90.0 → 9.0, Difficulty 60.0 → 6.0

## Files Modified

- ui/complete_task.py:
  - Removed all `* 10` scaling operations from value processing

  - Updated `current_expected_aversion` processing

  - Added comments clarifying values are stored on 0-100 scale

  - Includes safety checks (only fixes values > 100, ratio tolerance)

  - Non-interactive mode support (--yes flag)
