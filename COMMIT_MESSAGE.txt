Security: Add user_id validation and data isolation for import/export

This commit implements critical security fixes to ensure users can only
access and modify their own data through import/export functionality and
database queries.

Import/Export Security Fixes:
- Made user_id REQUIRED for all export functions (csv_export.py)
- Added user_id filtering to all user-specific tables in exports:
  * Tasks, TaskInstances, PopupTriggers, PopupResponses, Notes, SurveyResponses
- Added user_id parameter to all import functions (csv_import.py)
- Implemented CSV user_id validation: rows with mismatched user_id are
  skipped to prevent cross-user data editing
- All imported data is assigned to the logged-in user (overrides CSV values)
- Updated UI (settings_page.py) to pass user_id from get_current_user()

Data Isolation Improvements:
- Fixed database queries missing user_id filters in instance_manager.py:
  * _pause_instance_db, _start_instance_db, _resume_instance_db
  * _update_cancelled_instance_db, _add_prediction_to_instance_db
  * _delete_instance_db
  * _list_cancelled_instances_db() - Added user_id parameter and filter (returns empty if None)
  * _get_instance_db() - Added user_id parameter and filter (returns None if None)
  * _backfill_attributes_from_json_db() - Added user_id parameter and filter
  * _backfill_attributes_from_json_csv() - Added user_id parameter and filter
  * _get_previous_actual_averages_db() - Added user_id parameter and filter (returns empty if None)
  * _get_previous_actual_averages_csv() - Added user_id filter
  * _get_initial_aversion_db() - Added user_id parameter and filter (returns None if None)
  * _get_initial_aversion_csv() - Added user_id filter
  * _get_previous_aversion_average_db() - Added user_id parameter and filter (returns None if None)
  * _get_previous_aversion_average_csv() - Added user_id filter
  * _get_baseline_aversion_robust_db() - Added user_id parameter and filter (returns None if None)
  * _get_baseline_aversion_robust_csv() - Added user_id filter
  * _get_batch_baseline_aversions_db() - Added user_id parameter and filter (returns empty if None)
  * _get_batch_baseline_aversions_csv() - Added user_id parameter and filter
  * _get_baseline_aversion_sensitive_db() - Added user_id parameter and filter (returns None if None)
  * _get_baseline_aversion_sensitive_csv() - Added user_id filter
  * Updated public methods to accept and pass user_id: list_cancelled_instances(),
    get_instance(), get_previous_actual_averages(), get_initial_aversion(),
    get_previous_aversion_average(), get_baseline_aversion_robust(),
    get_batch_baseline_aversions(), get_baseline_aversion_sensitive(),
    backfill_attributes_from_json()
- Fixed notes_manager.py to require user_id for data isolation:
  * get_all_notes() - Added user_id parameter (returns empty list if None)
  * delete_note() - Added user_id parameter (returns False if None)
  * add_note() - Added user_id parameter (associates note with user)
  * _initialize_default_note() - Added user_id parameter (checks per-user)
  * Updated ui/notes_page.py to get user_id from get_current_user() and pass to all methods
- Fixed task_manager.py to require user_id for data isolation:
  * _update_task_db() - Now requires user_id (returns False if None, validates before query)
- Fixed _load_instances() in analytics.py to require user_id:
  * Added user_id parameter (required, returns empty DataFrame if None)
  * Includes user_id in initial query filter for both completed_only=True and False
  * Early validation returns empty DataFrame if user_id is None (before DB access)
  * Improves security and performance by filtering at database level
  * Updated cache key to include user_id for proper cache isolation
- Fixed productivity_tracker.py to require user_id for data isolation:
  * _get_completed_instances() - Added user_id filter (converts string to int for DB)
  * Returns empty DataFrame if user_id is None or invalid
- Updated UI pages to pass user_id:
  * ui/cancelled_tasks_page.py - Gets user_id from get_current_user() and passes to list_cancelled_instances()
  * ui/task_editing_manager.py - Gets user_id from get_current_user() and passes to list_cancelled_instances()
  * ui/notes_page.py - Gets user_id from get_current_user() and passes to all NotesManager methods
- Added user_id parameters to all public interface functions
- All database queries now filter by user_id before execution
- Functions return False/empty/None if user_id is None for security
- Migration functions (backfill_attributes_from_json) allow user_id=None for migration
  purposes but log security warnings

Security Guarantees:
- Users can only export their own data
- Users can only import data for themselves
- CSV user_id values are validated before import
- Mismatched user_id rows are skipped (preferring "few bad data points"
  over allowing cross-user data editing)
- No cross-user data access or modification possible

Testing Status:
⚠️  Both import/export security and data isolation changes require
    thorough testing before production deployment. The import feature
    remains disabled in the UI pending security audit completion.
