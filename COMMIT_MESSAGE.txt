fix(security): ensure all instance retrieval methods use logged-in user_id

Fix security warning where _get_instance_db() was called without user_id,
which could allow unauthorized data access. All instance retrieval methods
now require and use the logged-in user's ID for proper data isolation.

Changes:
- Fix append_instance_notes() in instance_manager.py to get current user
  * Import get_current_user() and validate user is logged in
  * Pass user_id to get_instance() for data isolation
- Fix mark_instance_as_edited() in task_editing_manager.py
  * Get current user before retrieving instance
  * Pass user_id to get_instance() to ensure user owns the instance
- Fix show_details() in dashboard.py
  * Get current user and validate authentication
  * Fix incorrect class method call (InstanceManager.get_instance() -> im.get_instance())
  * Pass user_id to get_instance() for security
- Fix three get_instance() calls in complete_task.py submit_completion()
  * Line 537: Instance validation check now uses current_user_id
  * Line 735: Edit mode instance retrieval now uses current_user_id
  * Line 802: Aversion slider update now uses current_user_id
- Fix get_previous_task_averages() to pass user_id through fallback chain
  * Method accepted user_id parameter but wasn't forwarding it to internal methods
  * Line 2574: Now passes user_id to _get_previous_task_averages_db()
  * Line 2576: Now passes user_id to _get_previous_task_averages_csv()
  * Line 2787: Exception handler fallback now passes user_id when falling back to CSV
  * Eliminates "[InstanceManager] WARNING: get_previous_task_averages_db() called without user_id" warning
  * Maintains data isolation in both normal and error fallback scenarios
- Fix analytics functions to require user_id for data access
  * Add _get_user_id() helper method to get user_id from authenticated session
  * Update core analytics functions to accept and use user_id parameter:
    - active_summary(), get_dashboard_metrics(), get_life_balance()
    - get_daily_work_volume_metrics(), calculate_time_tracking_consistency_score()
    - get_all_scores_for_composite(), calculate_daily_scores()
    - get_historical_daily_scores(), check_score_milestones()
    - calculate_weekly_progress_summary(), get_productivity_time_minutes()
    - get_relief_summary()
  * All functions now gracefully return empty/default data if user_id is None
  * Ensures monitored metrics require logged-in user_id or data doesn't load

Security Impact:
- Prevents potential unauthorized access to instances belonging to other users
- Ensures all database queries are properly filtered by user_id
- Eliminates "[InstanceManager] WARNING: _get_instance_db() called without user_id" warnings
- Eliminates "[InstanceManager] WARNING: get_previous_task_averages_db() called without user_id" warnings
- Analytics functions now require authentication for data access
- Monitored metrics will not load data without valid user_id
